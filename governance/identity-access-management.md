# 🔐 Identity & Access Management (IAM)

## Overview

XYZ Academy applies Identity and Access Management (IAM) practices to ensure that users can only access systems and resources required for their operational responsibilities.

The IAM framework helps reduce unauthorized access, improve accountability, and support organizational security operations.

---

## User Categories

The simulated environment includes multiple user groups with different access requirements.

| User Type | Access Scope |
|---|---|
| Students | LMS access, academic materials, personal records |
| Teachers | Course management, grading systems, communication services |
| Administrative Staff | Academic records and administrative systems |
| IT Administrators | Infrastructure management, backups, and monitoring systems |
| Security Operations Personnel | Security monitoring and incident handling activities |

---

## Access Control Principles

The academy environment applies multiple access control principles:

- Least Privilege
- Role-Based Access Control (RBAC)
- Separation of Duties (SoD)
- Need-to-Know Access
- Centralized Identity Management

These principles help limit excessive permissions and reduce operational security risks.

---

## Authentication Security

Authentication controls are implemented to improve account protection and reduce unauthorized access attempts.

### Implemented Controls

- Unique user accounts
- Strong password requirements
- Multi-factor authentication (MFA) for privileged accounts
- Password expiration policies
- Account lockout mechanisms after repeated failed login attempts

Privileged administrative systems require stronger authentication protections due to their higher operational sensitivity.

---

## Authorization Management

Authorization processes determine what resources each authenticated user can access within the environment.

Examples include:

- Students can access academic content but cannot modify records.
- Teachers can manage course materials and grading systems.
- Administrative staff can access reporting systems.
- IT administrators maintain infrastructure and backup services.

Authorization is controlled according to operational responsibilities and security policies.

---

## Account Lifecycle Management

User accounts follow a structured lifecycle management process.

### Account Provisioning
- New accounts are created according to user roles.
- Access approval is performed by authorized personnel.

### Access Modification
- Permissions are adjusted when responsibilities or departments change.

### Account Deactivation
- Accounts are disabled after graduation, resignation, or termination.

This process reduces the risk of orphaned accounts and unauthorized persistent access.

---

## Accountability & Monitoring

Authentication and access activities are logged and monitored to improve accountability and support incident investigations.

Examples include:

- Failed login attempts
- Administrative actions
- Password reset activities
- Access to sensitive systems
- Privileged account usage

Monitoring access activities supports operational security visibility and incident response readiness.
